Funcionamiento del Servicio

Los asientos contrbles tienen una estructura similar a la de una factura

Los pasos a seguir para completar una trasaccion son los siguientes:

  1. Enviar datos correspondientes al asiento contable que desea crear usando el formato que se muestra en la documentacion .
  2. Esa informacion sera registrada en nuestra base de datos, si cumple con todos lo requisitos que requiere el servicio
  3. Cuando el equipo de contabilidad autorize los asientos contables el sistema procede a hacer los moviemientos en las cuentas

Tokens de acceso

Para asegurar la seguridad del servicio y facilitar la integracion de diferentes modulos al servicio cada request debe tener en la cabecera una propiedad llamada token, si no cuenta con el token asignado a su equipo la API no le permite interactuar con el servicio.

Rutas de la API

Crear trasaccion

Para registrar un asientos contables debe enviar un Array con tantos asitentos contables como requiera registrar, a continuacion una request de ejemplo (Todos los demas datos seran autogenradaos haciendo uso de su token unico): 

            [
              {
                "Period": "2020-01",
                "Currency" : "USD",
                "Detail" : [
                  {
                    "Amount" : 100,
                    "LegerAccount" : 8,
                    "MovementType" : "DB"
                  },
                  {
                    "Amount" : 100,
                    "LegerAccount" : 8,
                    "MovementType" : "CR"
                  }
                ]
              }
            ]

Manejo de errores

Errores de autenticacion

En caso de que reciba como respuesta:

Confirme que este usando el token asignado a su grupo. Ver tokens

Errores de validacion

Lo que provoca el sigiente error:

A continuacion una tabla con posibles errores que valida el sistema al enviar los datos

Codigo del error Razon
1001 No hay ningun asiento contable.
1002 El formato del periodo no es correcto.(Debe ser xxxx-xx, una longitud de 7)
1003 El número de detalles es menor al permitido. Como mínimo deben ser 2 entradas de cuentas, una DB y otra CR.
1005 El asiento contable no está cuadrado. Diferentes montos en los detallles de credito y debito
1006 La cuenta contable con el ID que envia no existe. Revisar ID disponibles
1007 El Codigo de la moneda que envia no existe. (Monedas disponibles: DOP, USD, EUR )
